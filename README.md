LensBlaze – Photography & Creative Studio Website

LensBlaze is a modern, responsive website built for a photography and creative services brand. It is designed to showcase professional work, present services clearly, and enable smooth client interaction through a clean and minimal interface.

This project focuses on delivering a visual-first experience tailored for photographers, editors, and creative professionals.

Overview

LensBlaze serves as a digital presence for:

- Showcasing photography and creative work
- Presenting services in a structured format
- Engaging potential clients
- Supporting bookings and project inquiries

The website emphasizes simplicity, performance, and a premium visual feel.

Core Features

- Fully responsive design with a mobile-first approach
- Portfolio and selected-work presentation with grid-based layouts
- Clearly defined service sections
- Clean and minimal user interface
- Strong call-to-action flow for bookings
- Optimized layout for visual storytelling

Services Covered

LensBlaze provides creative services across multiple areas:

- Photography including portraits, events, and brand shoots
- Photo editing with professional retouching and color correction
- Video editing for reels, highlights, and content
- Graphic design for social media and branding assets

Project Structure

- index.html        → Main landing page
- links.html        → Links / legal / additional information page
- login.html        → Client login page (UI)
- styles.css        → Styling
- script.js         → Interactions (modals, navigation, form UI)
- assets/           → Fonts and optional local assets (images are loaded via public URLs)

Technology Stack

- Frontend: HTML, CSS, JavaScript (no framework)
- Styling: CSS (Grid, Flexbox)
- Hosting: Any static hosting (e.g., Vercel, Netlify) or a standard web server (Apache via XAMPP)
- Media Delivery: CDN-based image hosting (such as ImageKit)

Image Usage Guidelines

All images used in the project should:

- Be relevant to photography and creative work
- Be publicly accessible via URL
- Be optimized for performance and fast loading

Example format:

https://ik.imagekit.io/yourid/photography/image1.jpg

Note: Replace yourid and the image path with your actual ImageKit URL endpoint and uploaded file path. The example above will not load unless you own that ImageKit endpoint.

Running the Project

Option 1: Using a simple local server (recommended)

- From the project folder:
  - python -m http.server 8010
- Open:
  - http://localhost:8010/

Option 2: Using XAMPP (Apache)

- Place the project folder inside htdocs
- Start Apache in XAMPP
- Open (example):
  - http://localhost/LensBlaze/

Deployment

The project can be deployed to any static host. Ensure all images are using public URLs for proper rendering in production.

Customization

You can easily update:

- Portfolio images and galleries (image URLs)
- Service descriptions
- Packages and offerings
- Contact and booking sections

The structure is designed to allow quick modifications without affecting layout consistency.
